# Phase Model

The plugin assumes a phased website workflow:

1. `state-sync`
2. `project-brief`
3. `requirements-ia`
4. `brand-assets`
5. `frontend-build`
6. `backend-integrations`
7. `qa-release`
8. `handoff-continuity`

The phase order is intentional:

- product first
- structure second
- visual system third
- implementation fourth
- runtime behavior fifth
- verification sixth
- continuity last
